## 🛠️ Die Phasen im Detail

### 🔹 Phase 1 — Fundament (Monat 1–2)
*Fokus: Vertiefung von Python und Einstieg in digitale Zwillinge.*

*   **Python für Datenanalyse:** [Kaggle Learn](https://www.kaggle.com/learn) – Praxisnaher Einstieg in die Datenmanipulation.
*   **Fundamentals of Deep Learning:** [NVIDIA DLI](https://learn.nvidia.com) – Erste Schritte im KI-Bereich.
*   **3D & Digitale Zwillinge:**
    *   Installation von **NVIDIA Omniverse** und Durcharbeiten der *Getting Started Tour*.
    *   Verständnis der **USD (Universal Scene Description)** Grundkonzepte via [OpenUSD.org](https://openusd.org).

### 🔹 Phase 2 — Use-Case-Spezifisch (Monat 3–4)
*Fokus: Direkte Ausrichtung auf die KosterTI Verticals.*

| Use Case | Kurs / Ressource | Zertifikat |
| :--- | :--- | :--- |
| **Industrial Digital Twin** | NVIDIA DLI: Omniverse Developer Blueprint | DLI Certificate |
| **Synthetic Data Generation** | NVIDIA DLI: Synthetic Data Generation for Training AI | DLI Certificate |
| **3D Visualisierung / Quality Mgmt** | Blender (Open Source) + USD-Export Pipeline | — |
| **HPC Grundlagen** | Dell Technologies Learning: Introduction to HPC | *Optional* |

> 💡 **Empfohlene NVIDIA DLI Zertifikate:**
> *   *Applications of AI for Predictive Maintenance* (~8h) – Wichtig für Aerospace & Energy.
> *   *Building Real-Time Video AI Applications* – Wichtig für Synthetic Data & Qualitätsmanagement.

### 🔹 Phase 3 — Praktikumsstart-Ready (Monat 5–6)
*Fokus: Kollaboration, Schnittstellen und das erste eigene Projekt.*

*   **Git & GitHub:** Versionsverwaltung lernen mit [GitHub Skills](https://skills.github.com) (Branches, PRs, Issues).
*   **APIs:** Grundlagen von **REST APIs** und **JSON** für zukünftige Omniverse-Extensions.
*   **📋 Das Mini-Projekt (Meilenstein):**
    *   **Aufgabe:** Erstelle eine einfache Omniverse-Szene (z. B. eine Werkshalle mit 3 beweglichen Objekten).
    *   **Code:** Schreibe ein Python-Skript, das die Live-Positionen der Objekte ausliest und als JSON exportiert.
    *   *Ziel:* Nachweis von USD-Verständnis, Python-Praxis und API-Denkweise.
